### Отчет по лабораторной работе №1: "Цветовые модели"

#### **1. Цель работы**

Целью данной лабораторной работы являлось изучение основных цветовых моделей (RGB, CMYK, HSV), принципов их построения и преобразования цветов между ними. Для этого было необходимо разработать приложение с графическим интерфейсом, которое позволяет пользователю интерактивно задавать цвет в одной из моделей и автоматически видеть его представление в двух других.

#### **2. Реализация**

Приложение было разработано на языке Python с использованием стандартной библиотеки `tkinter` для создания графического интерфейса.

Структура приложения включает в себя:
*   Три основных блока для каждой цветовой модели: CMYK, RGB и HSV.
*   Интерактивные элементы управления: для каждого компонента цвета (например, R, G, B) предусмотрен ползунок и поле для ввода точного значения.
*   Палитра цветов: набор предустановленных цветов для быстрого выбора, с возможностью добавления пользовательских цветов.
*   Окно предпросмотра: цветная плашка, отображающая текущий выбранный цвет.

Ключевой механизм работы приложения — это система обратной связи. При изменении значения в любом из 10 полей (C, M, Y, K, R, G, B, H, S, V) вызывается соответствующая функция-обработчик. Эта функция:
1.  Считывает измененные значения из своей цветовой модели.
2.  Выполняет математическое преобразование в базовую модель RGB.
3.  Из модели RGB выполняет преобразование в оставшиеся две модели.
4.  Обновляет значения во всех полях и ползунках интерфейса.

Для предотвращения бесконечных циклов обновления (когда изменение RGB вызывает обновление CMYK, которое снова вызывает обновление RGB) был введен специальный флаг `updating`, который блокирует повторные вызовы обработчиков во время одного цикла обновления.

#### **3. Описание реализованных методов (цветовых моделей)**

**RGB (Red, Green, Blue)**
*   Принцип: Аддитивная цветовая модель, в которой новые цвета получаются путем смешивания трех базовых цветов — красного, зеленого и синего. Является основной моделью для устройств, излучающих свет, таких как мониторы и экраны смартфонов.
*   Координаты: Каждый компонент (R, G, B) задается целым числом от 0 до 255, где (0, 0, 0) — черный цвет (отсутствие свечения), а (255, 255, 255) — белый цвет (максимальная интенсивность всех компонентов).

**CMYK (Cyan, Magenta, Yellow, Key/Black)**
*   Принцип: Субтрактивная модель, используемая преимущественно в полиграфии. Цвета получаются путем вычитания определенных частей спектра из белого света (например, голубая краска поглощает красный цвет).
*   Координаты: Компоненты C, M, Y, K задаются в процентах от 0 до 100.
*   Преобразование из RGB:
    1.  Значения R, G, B нормализуются до диапазона.
    2.  Вычисляется ключевой цвет (Black, K) по формуле: `K = 1 - max(R, G, B)`.
    3.  Остальные компоненты вычисляются как:
        *   `C = (1 - R - K) / (1 - K)`
        *   `M = (1 - G - K) / (1 - K)`
        *   `Y = (1 - B - K) / (1 - K)`
    Использование отдельного черного канала (K) позволяет получать более глубокий черный цвет и экономить краски по сравнению со смешиванием C, M и Y.

**HSV (Hue, Saturation, Value)**
*   Принцип: Цилиндрическая модель, которая более интуитивно понятна для человека, так как она разделяет цвет на его основные характеристики.
*   Координаты:
    *   Hue (Цветовой тон): Сам цвет (красный, желтый, зеленый и т.д.). Задается в градусах от 0 до 360 на цветовом круге.
    *   Saturation (Насыщенность): "Чистота" или "сочность" цвета. Задается в процентах от 0 (оттенок серого) до 100 (максимально чистый цвет).
    *   Value (Яркость/Значение): Освещенность цвета. Задается в процентах от 0 (черный) до 100 (яркий, насыщенный цвет).
*   Преобразование: Для конвертации между RGB и HSV использовалась стандартная функция `rgb_to_hsv` из библиотеки `colorsys`, которая реализует стандартный алгоритм преобразования.

#### **4. Вывод**

В ходе лабораторной работы было успешно создано приложение, демонстрирующее взаимосвязь между цветовыми моделями RGB, CMYK и HSV. Реализованный функционал позволяет наглядно изучить, как изменение одного компонента цвета в одной модели влияет на его представление в других, что полностью соответствует поставленной задаче.
